<h2 id="1-network란">1. Network란?</h2>
<p><img alt="" src="https://velog.velcdn.com/images/sommnie/post/8f1d645a-ca99-4b70-af9e-5eaea3d62732/image.png" /></p>
<p><strong>네트워크(Network)</strong>는
두 대 이상의 컴퓨터나 기기가 서로 데이터를 주고받기 위해 연결된 시스템입니다.</p>
<p>이러한 네트워크는 정보를 효율적으로 공유하고 자원을 최적화하기 위한 수단으로 사용됩니다.
현대의 <strong>인터넷(Internet)</strong>은 이러한 네트워크들이 서로 연결된 거대한 집합체입니다.</p>
<h2 id="⚙️-2-network의-구조">⚙️ 2. Network의 구조</h2>
<p>네트워크는 여러 요소들로 구성되며, 이들 간의 상호작용을 통해 데이터 통신이 이루어집니다.
주요 구성 요소는 다음과 같습니다 👇</p>
<h3 id="노드node">노드(Node)</h3>
<ul>
<li>네트워크에 연결된 장치 또는 기기를 의미</li>
<li>데이터 송수신의 주체로 작동</li>
<li>예시: 컴퓨터, 스마트폰, 서버, 라우터, 스위치 등<h3 id="링크link">링크(Link)</h3>
</li>
<li>노드 간 데이터를 주고받을 수 있는 <strong>경로</strong></li>
<li>연결 형태:<ul>
<li><strong>물리적 연결</strong>: 유선 케이블, 광섬유 등</li>
<li><strong>논리적 연결</strong>: Wi-Fi, 블루투스와 같은 무선 네트워크<h3 id="프로토콜protocol">프로토콜(Protocol)</h3>
</li>
</ul>
</li>
<li>네트워크에서 데이터 통신을 위한 규칙과 표준</li>
<li>통신 과정, 데이터 형식, 오류 검출 등을 정의</li>
<li>주요 프로토콜 예시:<ul>
<li>TCP/IP: 인터넷의 기본 프로토콜</li>
<li>HTTP/HTTPS: 웹 통신용 프로토콜</li>
<li>FTP: 파일 전송 프로토콜<h3 id="패킷packet">패킷(Packet)</h3>
</li>
</ul>
</li>
<li>데이터를 작은 단위로 나눈 전송 단위</li>
<li>네트워크 전송 효율을 높이기 위해 사용</li>
<li>각 패킷에는 출발지/목적지 주소, 데이터, 오류 검출 정보가 포함</li>
<li>수신 측에서는 패킷을 다시 조립하여 원래 데이터로 복원<h2 id="3-network-구조의-예시">3. Network 구조의 예시</h2>
<h3 id="31-간단한-홈-네트워크-🏠">3.1 간단한 홈 네트워크 🏠</h3>
</li>
<li><strong>노드</strong>: PC, 스마트폰, 스마트 TV</li>
<li><strong>링크</strong>: Wi-Fi 라우터를 통한 무선 연결</li>
<li><strong>프로토콜</strong>: TCP/IP 기반 인터넷 접속<h3 id="32-기업용-네트워크-🏢">3.2 기업용 네트워크 🏢</h3>
</li>
<li><strong>노드</strong>: 사내 서버, 직원 PC, 프린터</li>
<li><strong>링크</strong>: 이더넷 케이블, 광섬유, VPN</li>
<li><strong>프로토콜</strong>: HTTP/HTTPS, SMTP(이메일 전송), FTP<h2 id="4-network의-중요성">4. Network의 중요성</h2>
</li>
<li><strong>효율적인 데이터 공유</strong>: 파일, 미디어, 메시지를 실시간으로 전송 가능</li>
<li><strong>자원 최적화</strong>: 여러 사용자가 프린터, 서버 등의 자원을 공유</li>
<li><strong>연결성 확대</strong>: 전 세계 사용자 및 시스템과 실시간 통신 가능</li>
</ul>
